个人建站指北（施工中）

2019-11-06
  • 教程
  • network

总览

信息时代，能有一个自己网站，总是一件十分酷炫的事。然而，即使是一个非常普通的个人博客站，其背后隐藏的各种技术仍然足够让普通人望而却步。即使是相关专业的学生，学起来相关的内容也不免踩一些坑，无谓的浪费许多时间。
所以今天在这里，本人打算从头到尾，把建站的整个原理，流程，个人花了很长时间踩过的各种坑，都给大家记录下来。如果能够因此帮到一个两个有相关需求的朋友，那就能体现到这篇博文的价值了。
注意，本篇文章着重介绍概念，理论以及推荐相关的工具，网站，作指路之效。具体工具的使用步骤，一部分后续可能会另外更新专门的文章；即使没有，有了这些关键词之后在互联网上应该也很容易找到相关资料，请善用搜索引擎。

原理

建造一个个人网站，我们需要一些什么东西呢？
这个问题，需要我们先简单了解网站运行的原理之后才能回答。
网站的原理是什么呢？当我们打开浏览器，输入一个域名，敲击回车，一个网页就出现在我们面前了。一切都显得那么简单直接。
然而，“输入一个域名并敲击回车”，和“网页出现在我们面前”，这两件事之间其实发生了很多事情。当我们输入了域名之后，浏览器是不知道这个域名对应的ip地址的，因此浏览器会将域名打包发送给一个特殊的服务器，我们称之为DNS（域名服务器）。DNS会查找自己的数据库，在找到该域名对应的服务器ip地址后，将这个ip地址返回给浏览器。浏览器再根据这个ip地址，向该网站真正的服务器发送请求。网站服务器收到请求后，将网页需要的文件再返回给浏览器，浏览器就能根据这些文件构筑出一个网站了。
也就是说，整个流程其实是：

  1. 输入域名
  2. 浏览器发送请求给DNS
  3. DNS返回网站服务器ip
  4. 浏览器发送请求给网站服务器
  5. 网站服务器返回文件给浏览器
  6. 浏览器根据这些文件构筑网站

当然，现在的网站上有非常多的内容，大流量的网站需要考虑的问题也更多，这个原理仅仅只是最基本的内容，主要是用来辅助我们理解我们需要的东西的。
从这个流程当中，可以看出，我们需要以下几个东西：

  1. 网站域名
  2. DNS域名解析服务
  3. 网站服务器

网站域名是由一些域名注册管理机构管理的，如果我们想要拥有一个自己的域名，就需要向域名注册商租用域名的使用权。比较有名的域名注册商之一是Godaddy，很多读者们比较熟悉的公司，例如谷歌，腾讯等等也都有提供域名注册服务。
网站服务器也是类似的处理方式，需要你去一些服务器租赁商那里租用服务器。真正的服务器往往大而笨重，花费会非常高。而对于大部分的用户来说，他们的网站很小，很多时候用不到那么多的流量和性能。因此，不少厂商推出了一种新的租赁服务器的方式，也就是VPS（虚拟私人服务器）。这种服务器并非真正的物理服务器，而是通过软件的手段分享了真正的服务器的资源，从而大幅度的降低了租赁服务器的门槛。比较出名的提供VPS服务的厂商有Vultr，DigitalOcean，Linode，如果你偏爱大厂的话，也可以选择谷歌，亚马逊。这种虚拟化的好处还有很多，在此不继续赘述，有兴趣的同学可以自行查阅相关资料拓展阅读。
除了真的去租用一个服务器以外，还有不少更懒的方式去托管个人网站，其中不少还是免费的。有些网站例如Netlify，不仅可以免费托管静态网站，还能顺便帮你解决域名解析的问题，可以说是个人网站的福音了。不过，各方面综合考量的话我则是推荐GitHub Pages，这个博客目前就是托管在GitHub Pages上的，具体的实现方式见后文。
那么域名解析服务呢？刚才我们提到过，浏览器需要DNS服务器的帮助才能把域名转化为真正的服务器ip地址，这就是域名解析服务。有一些域名注册商会顺带包揽域名解析服务，但如果你想使用自己另外找的域名解析服务，也是完全可以的。在域名注册商那里购买域名的时候会注册一个账户，登录这个账户就可以管理你的域名了。域名管理包括许多功能，其中一个必要的功能就是设置域名解析服务。

域名

域名，是平时人们所说的“网址”的专业称呼。我们平时所见到的域名经常以.com，.org，.cn等等结尾。这些域名的“小尾巴”其实有一个统一的称呼，叫做“顶级域名”。为什么说它“顶级”呢？举个例子，bilibili.com和google.com，同属于.com域名，.com是顶级域名，而bilibili.com和google.com就是所谓的“二级域名”。在域名注册商那里，我们一般能买到的，就是这种二级域名。顶级域名理论上有给网站粗略分类的功能，例如.com商用，.org非盈利组织，以及.cn，.uk等等国家域名。但是实际使用的时候并没有做非常严格的区分，除了一些特殊情况，例如.gov就属于美国政府。
以此类推，www.bilibili.com和app.bilibili.com就可以被称为“三级域名”。需要注意的是，虽然他们两个都从属于“bilibili.com”这个二级域名，但是这三个域名可以指向完全不同的三个服务器。有些同学可能会问了，我输入www.bilibili.com的时候，浏览器自动跳转到了bilibili.com，这难道不是说明这两个域名都是指向同一个服务器吗？
这就需要我们对于“域名解析”这件事情有更细节的了解了。域名解析本质上就是一个表格，每一行就是一个记录（record），左边是域名，中间是记录类型（record type），右边是其他域名或者服务器ip地址。记录类型常见的有两种，“A record”直接指向服务器ip地址，而“CNAME record”则是指向其他域名。以本博客的域名shao11.com为例：

域名 记录类型 域名/IP
shao11.com A record 222.222.222.222
blog.shao11.com A record 33.33.33.33
www.shao11.com CNAME @
me.shao11.com CNAME blog.shao11.com

在以上的表格中我们就能看出，shao11.com指向的是222.222.222.222，这是一个“A record”，代表着域名shao11.com所指向的服务器ip是222.222.222.222。同理，blog.shao11.com则指向的是另一个ip地址33.33.33.33。如果说到这里都很好懂的话，这个第三行的@可能会让你有些费解。“CNAME record”正如之前所说，是由一个域名指向另一个域名，第四行的me.shao11.com指向blog.shao11.com就是一个很好的例子。而在这里，@是一个约定俗成的符号，指的是“老大”shao11.com。

服务器

服务器本质上和PC（个人电脑）是同样的架构，都是由CPU，内存和硬盘组成。理论上，你可以利用家庭宽带用你自己的电脑架设宽带。然而，这样做有很多问题，其中最致命的一个就是家庭宽带分配的ip地址不是固定的。此外，个人电脑故障死机，更新自动重启等等风险都比租用服务器高得多，更重要的是，一旦网站的流量达到一定程度，家庭带宽也不一定负担的起。所以我们一般会向一些服务器提供商租赁服务器来架设我们的网站。
假设你已经在上述的某个网站租赁了一台服务器或者VPS，拿到了专属的ip地址，我们登录管理系统，修改“DNS record”让我们的域名指向这个ip地址，事情显得似乎大功告成了？我们打开浏览器，输入我们的域名，浏览器开始转圈，经过很长一段时间之后（也有可能是立刻），浏览器给你报了个错：ERR_CONNECTION_REFUSED(或者是别的错误，这个有可能和客户端使用浏览器以及服务器用的操作系统有关系)。
嘿呀，静下心来，你仔细一想，发现了一个问题：服务器只是一个电脑，换句话说，如果它上面没有运行相应的软件，那它和你家里那台小笔记本没有任何区别，并且你还摸不到它。
这就涉及到一些更加硬核的操作部分了，如果没有计算机相关的经验，并且只是想建一个普通的，没有特殊功能的静态网站的话，我更推荐各位跳过下面的部分，直接阅读托管相关的内容；但是，如果你并不想止步于只是建一个展示确定内容的网站，而是有交互功能，比如能够创建账户，存储用户信息的话，使用服务器是一个避不过的坎，那我建议你仔细阅读下面的内容。
如果要建造一个网站的话，服务器一般需要安装这些软件：

  • 操作系统（OS）：例如Linux，Windows Server
  • 网页服务器（Web Server）：例如Apache，Node.js
  • 脚本语言（Script）：例如PHP，Javascript
  • 数据库（Database）：例如MySQL，PostgreSQL

需要注意的是，操作系统在租赁VPS的时候一般就需要选择了。大部分的人们会选择Linux作为服务器的操作系统，因为Linux不仅开源免费，而且有大量的社群维护，安全性和稳定性都有保障。然而，Linux操作系统本身是非常精简的，所以一般人安装使用的是类似于“整合包”一样的东西，在Linux的世界里称为“发行版”。而在所有的发行版当中，Ubuntu是比较方便的一个，此后的文章默认服务器使用的是Ubuntu操作系统。
现在我们有了操作系统，就可以在操作系统上安装“网页服务器”软件了。这一块的软件有很多选择，其本质就是持续在服务器上运行，监听网页（http/https）请求，一旦收到，就把相应的文件传给用户。http和https在计算机上都有默认的端口，如果把服务器比作楼房，ip就是邮政地址，端口(port)就是楼房里房间的门牌号。http对应的默认端口是80，而https则是443。当然，这些只是默认的端口，你可以修改设置让网页服务器在任意端口监听，但是相应的，域名也需要在后面添加对应的端口，例如https://shao11.com:2048。
脚本语言则是帮助你的网页实现一些特殊的功能。例如，如果你想要写一个网页用来代替骰子roll点，你就可以用Javascript写一个生成随机数的function，然后用HTML/CSS写一个UI出来。脚本语言有非常多的用途和用法，并非这篇文章能概括的内容，如果感兴趣的话可以专门去学习。
至于数据库，对于个人建站的同学们来说，除非你是想建那种有很多内容和数据需要展示和更新的那种网站，例如steam游戏数据之类的，否则对于新手而言，这个部分基本上是不用考虑的。和脚本语言一样，数据库相关的内容也脱离了这篇文章能够包含的内容，感兴趣的同学需要专门去学习。
