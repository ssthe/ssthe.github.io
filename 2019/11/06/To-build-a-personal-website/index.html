个人建站指北（施工中）

2019-11-06
  • 教程
  • hexo

总览

信息时代，能有一个自己网站，总是一件十分酷炫的事。然而，即使是一个非常普通的个人博客站，其背后隐藏的各种技术仍然足够让普通人望而却步。即使是相关专业的学生，学起来相关的内容也不免踩一些坑，无谓的浪费许多时间。
所以今天在这里，本人打算从头到尾，把建站的整个原理，流程，个人花了很长时间踩过的各种坑，都给大家记录下来。如果能够因此帮到一个两个有相关需求的朋友，那就能体现到这篇博文的价值了。

原理

建造一个个人网站，我们需要一些什么东西呢？
这个问题，需要我们先简单了解网站运行的原理之后才能回答。
网站的原理是什么呢？当我们打开浏览器，输入一个域名，敲击回车，一个网页就出现在我们面前了。一切都显得那么简单直接。
然而，“输入一个域名并敲击回车”，和“网页出现在我们面前”，这两件事之间其实发生了很多事情。当我们输入了域名之后，浏览器是不知道这个域名对应的ip地址的，因此浏览器会将域名打包发送给一个特殊的服务器，我们称之为DNS（域名服务器）。DNS会查找自己的数据库，在找到该域名对应的服务器ip地址后，将这个ip地址返回给浏览器。浏览器再根据这个ip地址，向该网站真正的服务器发送请求。网站服务器收到请求后，将网页需要的文件再返回给浏览器，浏览器就能根据这些文件构筑出一个网站了。
也就是说，整个流程其实是：

  1. 输入域名
  2. 浏览器发送请求给DNS
  3. DNS返回网站服务器ip
  4. 浏览器发送请求给网站服务器
  5. 网站服务器返回文件给浏览器
  6. 浏览器根据这些文件构筑网站

当然，现在的网站上有非常多的内容，大流量的网站需要考虑的问题也更多，这个原理仅仅只是最基本的内容，主要是用来辅助我们理解我们需要的东西的。
从这个流程当中，可以看出，我们需要以下几个东西：

  1. 网站域名
  2. DNS域名解析服务
  3. 网站服务器

网站域名是由一些域名注册管理机构管理的，如果我们想要拥有一个自己的域名，就需要向域名注册商租用域名的使用权。比较有名的域名注册商之一是Godaddy，很多读者们比较熟悉的公司，例如谷歌，腾讯等等也都有提供域名注册服务。
网站服务器也是类似的处理方式，需要你去一些服务器租赁商那里租用服务器。真正的服务器往往大而笨重，花费会非常高。而对于大部分的用户来说，他们的网站很小，很多时候用不到那么多的流量和性能。因此，不少厂商推出了一种新的租赁服务器的方式，也就是VPS（虚拟私人服务器）。这种服务器并非真正的物理服务器，而是通过软件的手段分享了真正的服务器的资源，从而大幅度的降低了租赁服务器的门槛。比较出名的提供VPS服务的厂商有Vultr，DigitalOcean，Linode，如果你偏爱大厂的话，也可以选择谷歌，亚马逊。这种虚拟化的好处还有很多，在此不继续赘述，有兴趣的同学可以自行查阅相关资料拓展阅读。
除了真的去租用一个服务器以外，还有不少更懒的方式去托管个人网站，其中不少还是免费的。有些网站例如Netlify，不仅可以免费托管静态网站，还能顺便帮你解决域名解析的问题，可以说是个人网站的福音了。不过，各方面综合考量的话我则是推荐GitHub Pages，这个博客目前就是托管在GitHub Pages上的，具体的实现方式见后文。
那么域名解析服务呢？刚才我们提到过，浏览器需要DNS服务器的帮助才能把域名转化为真正的服务器ip地址。那么当我们建造一个个人网站的时候，就需要让DNS服务器知道我们的域名对应的服务器地址是哪个。有一些域名注册商会顺带包揽域名解析服务，但如果你想使用自己另外找的域名解析服务，也是完全可以的。在域名注册商那里购买域名的时候会注册一个账户，登录这个账户就可以管理你的域名了，其中一个必要的功能就是设置域名解析服务。其它更具体的知识和设置呢，我们喝杯茶，接着说。
